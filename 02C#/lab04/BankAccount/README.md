## Описание классов

### Класс Currency (Валюта)

**Назначение:** Представляет валюту с кодом и курсом обмена к базовой валюте (рублю).

**Приватные поля:**
- `string code` - код валюты (RUR, INR, USD)
- `double exchangeRate` - курс обмена к рублю

**Публичные свойства:**
- `string Code { get; }` - получить код валюты (только чтение)
- `double ExchangeRate { get; }` - получить курс обмена (только чтение)

**Публичные методы:**
- `Currency(string code, double exchangeRate)` - конструктор
- `double ConvertToBase(double amount)` - конвертировать сумму в рубли
- `double ConvertFromBase(double amount)` - конвертировать сумму из рублей в данную валюту
- `string ToString()` - строковое представление валюты

**Пример использования:**
```csharp
Currency usd = new Currency("USD", 95.0);
double rublesFromUsd = usd.ConvertToBase(100);  // 100 USD = 9500 RUR
double usdFromRubles = usd.ConvertFromBase(9500);  // 9500 RUR = 100 USD
```

### Класс Account (Счет)

**Назначение:** Представляет банковский счет с балансом в базовой валюте и операциями зачисления/снятия.

**Приватные поля:**
- `double balance` - баланс счета в рублях (базовая валюта)

**Публичные методы:**
- `Account()` - конструктор (создает счет с нулевым балансом)
- `double GetBalance()` - получить баланс в рублях
- `double GetBalanceInCurrency(Currency currency)` - получить баланс в заданной валюте
- `bool Deposit(double amount, Currency currency)` - зачислить средства
  - Возвращает `true` при успехе, `false` при ошибке (отрицательная сумма)
- `bool Withdraw(double amount, Currency currency)` - снять средства
  - Возвращает `true` при успехе, `false` при недостатке средств или отрицательной сумме

**Пример использования:**
```csharp
Account account = new Account();

Currency usd = new Currency("USD", 95.0);
account.Deposit(100, usd);  // Зачислить 100 USD (9500 RUR)

Currency inr = new Currency("INR", 0.90);
double balanceInr = account.GetBalanceInCurrency(inr);  // Баланс в рупиях: 10555.56 INR

bool success = account.Withdraw(50, usd);  // Снять 50 USD (4750 RUR)
// Остаток: 4750 RUR
```

## Принципы инкапсуляции

Все поля классов объявлены как `private`, что обеспечивает:
- Защиту данных от прямого изменения извне
- Контроль доступа через публичные методы
- Возможность валидации данных при изменении

Все манипуляции с данными происходят через публичные методы, которые:
- Проверяют корректность входных данных
- Выполняют необходимые преобразования
- Обеспечивают согласованность состояния объекта

## Ключевые особенности реализации

1. **Централизованное хранение в базовой валюте**  
   Все операции хранятся в рублях, что упрощает логику и избегает ошибок округления

2. **Автоматическая конвертация**  
   При зачислении/снятии сумма автоматически конвертируется через класс Currency

3. **Валидация данных**  
   - Проверка положительности суммы
   - Проверка достаточности средств при снятии
   - Обработка некорректного ввода

4. **Гибкое отображение**  
   Пользователь может выбрать любую валюту для отображения баланса без изменения фактического состояния счета

5. **Расширяемость**  
   Легко добавить новые валюты через массив `Currency[]`
