# Лабораторная работа №4: Разработка классов в C#

## Задание
**Вариант 10: "Банковский счет"**

Разработать класс `Currency` для работы с валютами (INR, USD) и класс `Account` для управления банковским счетом с операциями зачисления и снятия средств.

## Цель работы
Освоение принципов объектно-ориентированного программирования в C#: создание классов, инкапсуляция данных, использование свойств и методов для работы с объектами.

## Структура проекта

### BankAccount (консольная версия)
Первоначальная реализация с консольным интерфейсом.

### BankAccountGUI (графический интерфейс)
Графическая версия приложения на базе Avalonia UI.

#### Основные классы

**Currency.cs** - класс для работы с валютами:
- `Code` - код валюты (RUR, INR, USD)
- `ExchangeRate` - курс обмена относительно базовой валюты (RUR)
- `ConvertToBase()` - конвертация в базовую валюту
- `ConvertFromBase()` - конвертация из базовой валюты

**Account.cs** - класс банковского счета:
- `balance` - баланс счета в базовой валюте (RUR)
- `GetBalance()` - получить баланс
- `GetBalanceInCurrency()` - получить баланс в заданной валюте
- `Deposit()` - зачислить средства
- `Withdraw()` - снять средства

#### Интерфейс

Приложение имеет две функциональные зоны:

**Зона 1: Состояние счета**
- Отображение текущего баланса
- Выбор валюты для отображения (RUR/INR/USD)

**Зона 2: Транзакция**
- Ввод суммы операции
- Выбор валюты транзакции
- Кнопки "Зачислить" и "Снять"
- Отображение сообщений об успехе/ошибке

## Запуск

### Консольная версия
```bash
cd BankAccount
dotnet run
```

### Графическая версия
```bash
cd BankAccountGUI
dotnet run
```

## Особенности реализации

- Все операции выполняются через базовую валюту (RUR)
- Курсы валют: INR = 0.90 RUR, USD = 95.0 RUR
- Проверка достаточности средств при снятии
- Валидация введенных данных
- Современный кроссплатформенный GUI на Avalonia

## Вывод

В ходе выполнения лабораторной работы были освоены основы объектно-ориентированного программирования в C#. Разработаны два класса: `Currency` для работы с валютами и `Account` для управления банковским счетом.

Получен практический опыт применения принципов ООП: инкапсуляции через использование модификаторов доступа (private, public), создания свойств для безопасного доступа к данным, реализации методов для выполнения операций над объектами.

Реализованы две версии приложения: консольная для отработки базовой логики и графическая на базе Avalonia UI для создания современного кроссплатформенного интерфейса. Графическая версия демонстрирует разделение интерфейса на функциональные зоны и обработку событий пользовательского интерфейса.

Полученные навыки работы с классами и объектами являются фундаментальными для дальнейшего изучения механизмов наследования и полиморфизма в следующих лабораторных работах.
