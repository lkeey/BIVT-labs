## Теоретическая часть

### Условный оператор if-else

Условный оператор `if` позволяет выполнять различные действия в зависимости от выполнения условия.

#### Синтаксис

```csharp
if (условие)
{
    // Выполняется, если условие истинно
}
else
{
    // Выполняется, если условие ложно
}
```

#### Примеры использования

**Пример 1: Простое разветвление**

```csharp
double x = 5.0;
if (x >= 0)
{
    Console.WriteLine("Число неотрицательное");
}
else
{
    Console.WriteLine("Число отрицательное");
}
```

**Пример 2: Вложенные условия**

```csharp
int score = 85;
if (score >= 90)
{
    Console.WriteLine("Отлично (5)");
}
else if (score >= 75)
{
    Console.WriteLine("Хорошо (4)");
}
else if (score >= 60)
{
    Console.WriteLine("Удовлетворительно (3)");
}
else
{
    Console.WriteLine("Неудовлетворительно (2)");
}
```

#### Тернарный оператор

Краткая форма условного оператора для простых случаев:

```csharp
int max = (a > b) ? a : b;
string result = (x >= 0) ? "Положительное" : "Отрицательное";
```

### Множественный выбор switch

Оператор `switch` используется для выбора одного из множества вариантов на основе значения переменной.

#### Синтаксис

```csharp
switch (переменная)
{
    case значение1:
        // Код для случая 1
        break;
    case значение2:
        // Код для случая 2
        break;
    default:
        // Код по умолчанию
        break;
}
```

#### Примеры использования

**Пример 1: Выбор действия по коду**

```csharp
int choice = 2;
switch (choice)
{
    case 1:
        Console.WriteLine("Площадь квадрата");
        break;
    case 2:
        Console.WriteLine("Площадь круга");
        break;
    case 3:
        Console.WriteLine("Площадь треугольника");
        break;
    default:
        Console.WriteLine("Неверный выбор");
        break;
}
```

**Пример 2: Switch-выражения (C# 8.0+)**

```csharp
string figureName = figureType switch
{
    1 => "Квадрат",
    2 => "Круг",
    3 => "Треугольник",
    _ => "Неизвестная фигура"
};
```

### Логические операторы

Логические операторы используются для построения сложных условий.

#### Основные операторы

| Оператор | Название | Описание | Пример |
|----------|----------|----------|--------|
| `&&` | Логическое И (AND) | Истина, если оба условия истинны | `(x > 0) && (x < 10)` |
| `\|\|` | Логическое ИЛИ (OR) | Истина, если хотя бы одно условие истинно | `(x < 0) \|\| (x > 10)` |
| `!` | Логическое НЕ (NOT) | Инвертирует значение | `!(x > 0)` |

#### Операторы сравнения

| Оператор | Описание | Пример |
|----------|----------|--------|
| `==` | Равно | `x == 5` |
| `!=` | Не равно | `x != 0` |
| `>` | Больше | `x > 10` |
| `<` | Меньше | `x < 5` |
| `>=` | Больше или равно | `x >= 0` |
| `<=` | Меньше или равно | `x <= 100` |

#### Примеры сложных условий

```csharp
// Проверка вхождения в диапазон
if (x >= -1 && x <= 1)
{
    Console.WriteLine("x в диапазоне [-1, 1]");
}

// Проверка принадлежности точки треугольнику
if (y >= 0 && y <= 1 + x && y <= 1 - x)
{
    Console.WriteLine("Точка внутри треугольника");
}

// Проверка на неуспевающего студента
bool isFailing = (grade1 == 2) || (grade2 == 2) || (grade3 == 2) || (grade4 == 2);
```

#### Приоритет операций

1. `!` (NOT) - наивысший приоритет
2. Операторы сравнения (`>`, `<`, `>=`, `<=`, `==`, `!=`)
3. `&&` (AND)
4. `||` (OR) - наименьший приоритет

### Обработка потока данных

#### Случай 1: Количество данных известно

Когда количество данных известно заранее, используется цикл `for` с фиксированным числом итераций:

```csharp
Console.Write("Введите количество студентов: ");
int n = int.Parse(Console.ReadLine());

for (int i = 1; i <= n; i++)
{
    Console.WriteLine($"Студент {i}:");
    // Обработка данных студента
}
```

#### Случай 2: Количество данных неизвестно

Когда количество данных заранее неизвестно, используется цикл с условием и специальное значение для завершения ввода:

```csharp
while (true)
{
    Console.Write("Введите значение (или -1 для завершения): ");
    double value = double.Parse(Console.ReadLine());
    
    if (value == -1)  // Признак конца ввода
        break;
    
    // Обработка значения
}
```

**Выбор признака конца ввода:**
- Если все данные положительные → используем отрицательное число
- Если данные целые → можем использовать специальное число вне диапазона
- Можно использовать специальную строку (например, "exit", "quit")

#### Особенности ввода нескольких значений

При вводе нескольких значений для одного элемента (например, 4 оценки студента), одно из значений выбирается как контрольное:

```csharp
do
{
    Console.Write("Введите 4 оценки (или -1 для завершения): ");
    string input = Console.ReadLine();
    
    if (input.Trim() == "-1")
        break;
    
    string[] parts = input.Split(' ');
    int[] grades = Array.ConvertAll(parts, int.Parse);
    
    // Обработка оценок
} while (true);
```

#### Задача 1: Точка на окружности

**Условие:** На плоскости расположена окружность радиусом r = 2 с центром в начале координат. Ввести заданные координаты точки и определить, лежит ли она на окружности. Точка с координатами (x, y) лежит на окружности радиусом r, если |x² + y² - r²| ≤ 10⁻³.

**Математическое обоснование:**

Расстояние от точки (x, y) до центра окружности (0, 0) вычисляется по формуле:

```
d = √(x² + y²)
```

Точка лежит на окружности, если d = r (с учетом погрешности).

**Решение:**

```csharp
const double r = 2.0;
const double eps = 1e-3;

double distanceSquared = x * x + y * y;
double rSquared = r * r;
double difference = Math.Abs(distanceSquared - rSquared);
bool onCircle = difference <= eps;
```

**Тестовые данные:**

| Точка (x, y) | Расстояние | \|d² - r²\| | Результат |
|--------------|------------|-------------|-----------|
| (0.0, 2.0) | 2.0000 | 0.000000 | На окружности |
| (1.5, 0.7) | 1.6583 | 1.250900 | Вне окружности |
| (1.0, 1.0) | 1.4142 | 2.000000 | Вне окружности |
| (3.0, 0.0) | 3.0000 | 5.000000 | Вне окружности |

**Анализ:** Только точка (0, 2) находится на окружности, так как её расстояние от центра точно равно радиусу 2.

#### Задача 2: Точка в треугольнике

**Условие:** Определить, лежит ли заданная точка внутри или вне треугольника с вершинами в точках (-1, 0), (1, 0), (0, 1).

**Математическое обоснование:**

Треугольник ограничен тремя прямыми:
1. Основание: y = 0 (ось абсцисс)
2. Левая сторона: y = 1 + x (при x < 0)
3. Правая сторона: y = 1 - x (при x ≥ 0)

Точка внутри треугольника, если выполняются условия:
- y ≥ 0 (выше основания)
- y ≤ 1 + x (слева от левой границы, при x < 0)
- y ≤ 1 - x (слева от правой границы, при x ≥ 0)

**Решение:**

```csharp
bool IsInsideTriangle(double x, double y)
{
    if (y < 0) return false;  // Ниже основания
    
    if (x < 0)
        return y <= 1 + x;  // Левая граница
    else
        return y <= 1 - x;  // Правая граница
}
```

**Тестовые данные:**

| Точка (x, y) | Описание | Результат |
|--------------|----------|-----------|
| (0.0, 0.5) | Центр треугольника | Внутри |
| (-0.5, 0.3) | Внутри слева | Внутри |
| (0.5, 0.3) | Внутри справа | Внутри |
| (0.0, 0.0) | Вершина B | Внутри |
| (0.0, 1.0) | Вершина C | Внутри |
| (1.5, 0.5) | Справа от треугольника | Снаружи |

#### Задача 10: Студенты без "2" и "3"

**Условие:** В группе n студентов. Каждый получил 4 оценки. Подсчитать число студентов, не имеющих "2" и "3".

**Алгоритм:**

1. Ввести количество студентов n
2. Для каждого студента:
   - Ввести 4 оценки
   - Проверить, есть ли среди них "2" или "3"
   - Если нет, увеличить счетчик
3. Вывести результат

**Решение:**

```csharp
int studentsWithoutLowGrades = 0;

for (int i = 1; i <= n; i++)
{
    // Ввод оценок
    int[] grades = ReadGrades();
    
    // Проверка наличия "2" или "3"
    bool hasLowGrades = false;
    foreach (int grade in grades)
    {
        if (grade == 2 || grade == 3)
        {
            hasLowGrades = true;
            break;
        }
    }
    
    if (!hasLowGrades)
        studentsWithoutLowGrades++;
}

Console.WriteLine($"Студентов без '2' и '3': {studentsWithoutLowGrades}");
```

#### Задача 11: Неуспевающие и средний балл

**Условие:** В группе n студентов. Каждый сдал 4 экзамена. Подсчитать число неуспевающих студентов (имеющих хотя бы одну "2") и средний балл группы.

**Алгоритм:**

1. Ввести количество студентов n
2. Инициализировать счетчики: неуспевающих, суммы оценок
3. Для каждого студента:
   - Ввести 4 оценки
   - Проверить наличие "2"
   - Добавить оценки к общей сумме
4. Вычислить средний балл: сумма/(n×4)
5. Вывести результаты

**Решение:**

```csharp
int failingStudents = 0;
int totalGradesSum = 0;

for (int i = 1; i <= n; i++)
{
    int[] grades = ReadGrades();
    
    bool isFailing = false;
    int studentSum = 0;
    
    foreach (int grade in grades)
    {
        studentSum += grade;
        if (grade == 2)
            isFailing = true;
    }
    
    if (isFailing)
        failingStudents++;
    
    totalGradesSum += studentSum;
}

double groupAverage = (double)totalGradesSum / (n * 4);

Console.WriteLine($"Неуспевающих студентов: {failingStudents}");
Console.WriteLine($"Средний балл группы: {groupAverage:F2}");
```

#### Задача 12: Вычисление площадей (множественный выбор)

**Условие:** Ввести n значений r, вычислить по выбору:
1. Площадь квадрата: S = r²
2. Площадь круга: S = πr²
3. Площадь равностороннего треугольника: S = (r²√3)/4

**Алгоритм:**

1. Ввести количество значений n
2. Выбрать тип фигуры (1, 2 или 3)
3. Для каждого значения r:
   - Ввести r
   - Вычислить площадь по выбранной формуле
   - Вывести результат

**Решение:**

```csharp
double CalculateArea(int figureType, double r)
{
    switch (figureType)
    {
        case 1:
            return r * r;  // Квадрат
        
        case 2:
            return Math.PI * r * r;  // Круг
        
        case 3:
            return (r * r * Math.Sqrt(3)) / 4;  // Треугольник
        
        default:
            throw new ArgumentException("Неверный тип фигуры");
    }
}
```

## Контрольные вопросы

### 1. Что такое разветвление? Какой оператор используется для организации разветвлений?

**Ответ:**

**Разветвление** — это структура алгоритма, в которой выполнение программы идет по одному из нескольких возможных путей в зависимости от выполнения некоторого условия.

Для организации разветвлений в C# используется **условный оператор `if`**:

```csharp
if (условие)
{
    // Ветвь ИСТИНА
}
else
{
    // Ветвь ЛОЖЬ
}
```

**Виды разветвлений:**

1. **Полное разветвление** — содержит обе ветви (if-else)
2. **Неполное разветвление (обход)** — содержит только одну ветвь (if без else)
3. **Множественное разветвление** — выбор из нескольких вариантов (if-else if-else или switch)

**Примеры:**

```csharp
// Полное разветвление
if (x > 0)
    Console.WriteLine("Положительное");
else
    Console.WriteLine("Неположительное");

// Обход (неполное разветвление)
if (balance < 0)
    Console.WriteLine("Внимание: отрицательный баланс!");

// Множественное разветвление
if (score >= 90)
    grade = "5";
else if (score >= 75)
    grade = "4";
else if (score >= 60)
    grade = "3";
else
    grade = "2";
```

---

### 2. Обход и его реализация на языке C#

**Ответ:**

**Обход** — это частный случай разветвления, когда одна из ветвей не содержит никаких действий. При невыполнении условия программа просто продолжает выполнение следующего оператора.

**Реализация:**

```csharp
if (условие)
{
    // Действия выполняются только если условие истинно
}
// Продолжение программы
```

**Примеры использования:**

```csharp
// Проверка на ошибку
if (denominator == 0)
{
    Console.WriteLine("Ошибка: деление на ноль!");
    return;
}

// Валидация данных
if (age < 0 || age > 120)
{
    Console.WriteLine("Некорректный возраст");
    return;
}

// Дополнительная обработка при условии
if (isVIP)
{
    discount += 10;
    Console.WriteLine("Применена VIP-скидка");
}
```

**Блок-схема обхода:**

```
     ┌──────────┐
     │ Условие? │
     └────┬─────┘
       ДА│
     ┌────▼─────┐
     │ Действие │
     └────┬─────┘
          │◄── НЕТ
     ┌────▼────────┐
     │ Продолжение │
     └─────────────┘
```

### 3. Множественный выбор и его реализация на языке C#

**Ответ:**

**Множественный выбор** — это обобщение разветвления, когда в зависимости от значения переменной выполняется одно из нескольких действий.

**Реализация с помощью if-else if:**

```csharp
if (choice == 1)
{
    Console.WriteLine("Вариант 1");
}
else if (choice == 2)
{
    Console.WriteLine("Вариант 2");
}
else if (choice == 3)
{
    Console.WriteLine("Вариант 3");
}
else
{
    Console.WriteLine("Неверный выбор");
}
```

**Реализация с помощью switch:**

```csharp
switch (choice)
{
    case 1:
        Console.WriteLine("Вариант 1");
        break;
    case 2:
        Console.WriteLine("Вариант 2");
        break;
    case 3:
        Console.WriteLine("Вариант 3");
        break;
    default:
        Console.WriteLine("Неверный выбор");
        break;
}
```

**Современный switch-expression (C# 8.0+):**

```csharp
string message = choice switch
{
    1 => "Вариант 1",
    2 => "Вариант 2",
    3 => "Вариант 3",
    _ => "Неверный выбор"
};
```

**Преимущества switch:**
- Более читаемый код при большом количестве вариантов
- Компилятор может оптимизировать выполнение
- Удобен для работы с перечислениями (enum)

**Пример из практики (Задача 12):**

```csharp
double CalculateArea(int figureType, double r)
{
    switch (figureType)
    {
        case 1:
            return r * r;  // Квадрат
        case 2:
            return Math.PI * r * r;  // Круг
        case 3:
            return (r * r * Math.Sqrt(3)) / 4;  // Треугольник
        default:
            throw new ArgumentException("Неверный тип");
    }
}
```

### 4. Логические переменные, логические операторы, логические выражения и их использование в программах

**Ответ:**

#### Логические переменные (bool)

Тип `bool` может принимать только два значения: `true` (истина) или `false` (ложь).

```csharp
bool isStudent = true;
bool hasDiscount = false;
bool isAdult = age >= 18;
```

#### Логические операторы

| Оператор | Название | Таблица истинности |
|----------|----------|-------------------|
| `&&` | Логическое И (AND) | `true && true = true`<br>`true && false = false`<br>`false && false = false` |
| `\|\|` | Логическое ИЛИ (OR) | `true \|\| false = true`<br>`false \|\| false = false` |
| `!` | Логическое НЕ (NOT) | `!true = false`<br>`!false = true` |

#### Логические выражения

Логическое выражение — это выражение, результатом которого является `true` или `false`.

```csharp
// Простые выражения
x > 0
age >= 18
name == "Admin"

// Сложные выражения
(x > 0) && (x < 10)  // x в диапазоне (0, 10)
(grade == 2) || (grade == 3)  // оценка 2 или 3
!(x > 0)  // x неположительное
```

#### Использование в программах

**Пример 1: Проверка условий доступа**

```csharp
bool isAdult = age >= 18;
bool hasPermission = role == "Admin" || role == "Manager";
bool canAccess = isAdult && hasPermission;

if (canAccess)
    Console.WriteLine("Доступ разрешен");
```

**Пример 2: Проверка принадлежности точки области (из Задачи 2)**

```csharp
bool insideTriangle = (y >= 0) && 
                      ((x < 0 && y <= 1 + x) || 
                       (x >= 0 && y <= 1 - x));
```

**Пример 3: Проверка успеваемости (из Задачи 10)**

```csharp
bool hasLowGrades = (grade1 == 2 || grade1 == 3) ||
                    (grade2 == 2 || grade2 == 3) ||
                    (grade3 == 2 || grade3 == 3) ||
                    (grade4 == 2 || grade4 == 3);

if (!hasLowGrades)
    studentsWithGoodGrades++;
```

#### Короткое замыкание (Short-circuit evaluation)

Операторы `&&` и `||` используют короткое замыкание:

```csharp
// Второе условие не вычисляется, если первое false
if (array != null && array.Length > 0)
{
    // Безопасно использовать array
}

// Второе условие не вычисляется, если первое true
if (x == 0 || y / x > 10)
{
    // Избегаем деления на ноль
}
```

### 5. Организация обработки потока данных, количество которых заранее не известно. Использование специального значения

**Ответ:**

Когда количество данных заранее неизвестно, используются циклы с условием (`while`, `do-while`) и специальное значение для обозначения конца ввода.

#### Выбор специального значения (признака конца ввода)

**Принципы выбора:**
1. Значение должно быть того же типа, что и данные
2. Значение не должно встречаться в реальных данных
3. Должно быть легко вводить пользователю

**Примеры:**

| Тип данных | Диапазон значений | Признак конца |
|------------|-------------------|---------------|
| Положительные числа | > 0 | -1 или 0 |
| Целые числа | любые | специальное число вне диапазона |
| Оценки | 2-5 | -1 |
| Строки | любые | "exit", "quit", "" |

#### Реализация с циклом while

```csharp
while (true)
{
    Console.Write("Введите значение (или -1 для завершения): ");
    double value = double.Parse(Console.ReadLine());
    
    if (value == -1)  // Проверка признака конца
        break;
    
    // Обработка значения
    sum += value;
    count++;
}
```

#### Реализация с циклом do-while

```csharp
double value;
do
{
    Console.Write("Введите значение (или -1 для завершения): ");
    value = double.Parse(Console.ReadLine());
    
    if (value != -1)
    {
        sum += value;
        count++;
    }
} while (value != -1);
```

#### Пример из практики (Задача 10, Уровень III)

```csharp
int totalStudents = 0;
int studentsWithoutLowGrades = 0;

while (true)
{
    Console.Write("Введите 4 оценки (или '-1' для завершения): ");
    string input = Console.ReadLine();
    
    // Проверка признака конца
    if (input.Trim() == "-1")
    {
        Console.WriteLine("Ввод завершен.");
        break;
    }
    
    // Обработка оценок
    string[] parts = input.Split(' ');
    if (parts.Length != 4) continue;
    
    totalStudents++;  // Динамический подсчет количества
    
    // ... проверка оценок ...
}

Console.WriteLine($"Всего студентов: {totalStudents}");
```

#### Преимущества метода

1. **Гибкость** — пользователь сам определяет, сколько данных ввести
2. **Удобство** — не нужно заранее подсчитывать количество
3. **Универсальность** — подходит для интерактивных программ

#### Недостатки

1. Нужно выбрать подходящее специальное значение
2. Возможны ошибки при случайном вводе признака конца
3. Требуется явная проверка в каждой итерации

### 6. Особенности организации ввода, если каждая порция данных включает несколько значений

**Ответ:**

При вводе нескольких связанных значений (например, 4 оценки студента) для организации цикла выбирается **одно значение в качестве контрольного** для проверки признака конца ввода.

#### Стратегия: Первое значение как контрольное

```csharp
do
{
    Console.Write("Введите первую оценку (или -1 для завершения): ");
    int grade1 = int.Parse(Console.ReadLine());
    
    if (grade1 == -1)
        break;
    
    Console.Write("Введите вторую оценку: ");
    int grade2 = int.Parse(Console.ReadLine());
    
    Console.Write("Введите третью оценку: ");
    int grade3 = int.Parse(Console.ReadLine());
    
    Console.Write("Введите четвертую оценку: ");
    int grade4 = int.Parse(Console.ReadLine());
    
    // Обработка всех четырех оценок
} while (true);
```

#### Стратегия: Все значения в одной строке

Более удобный вариант — вводить все значения в одной строке через пробел:

```csharp
while (true)
{
    Console.Write("Введите 4 оценки через пробел (или '-1' для завершения): ");
    string input = Console.ReadLine();
    
    // Проверка признака конца
    if (input.Trim() == "-1")
        break;
    
    // Разбор строки
    string[] parts = input.Split(' ', StringSplitOptions.RemoveEmptyEntries);
    
    if (parts.Length != 4)
    {
        Console.WriteLine("Ошибка: нужно ввести ровно 4 оценки");
        continue;

    // Преобразование в числа
    int[] grades = new int[4];
    for (int i = 0; i < 4; i++)
    {
        if (!int.TryParse(parts[i], out grades[i]))
        {
            Console.WriteLine("Ошибка: некорректное значение");
            continue;
        }
    }
    
    // Обработка оценок
    ProcessGrades(grades);
}
```

### 7. Что такое множественный выбор? Как он реализуется?

**Ответ:**

**Множественный выбор** — это структура алгоритма, позволяющая выбрать одно из нескольких действий на основе значения некоторой переменной (селектора).

#### Схема множественного выбора

```
     ┌──────────┐
     │ Селектор │
     └────┬─────┘
          │
    ┌─────┼─────┬─────┐
    │     │     │     │
  ┌─▼─┐ ┌─▼─┐ ┌─▼─┐ ┌─▼───┐
  │ 1 │ │ 2 │ │ 3 │ │ else│
  └─┬─┘ └─┬─┘ └─┬─┘ └─┬───┘
    │     │     │     │
    └─────┼─────┴─────┘
          │
     ┌────▼────┐
     │Дальше...│
     └─────────┘
```

#### Реализация в C#

**Способ 1: Каскад if-else if**

```csharp
if (choice == 1)
{
    // Действие 1
}
else if (choice == 2)
{
    // Действие 2
}
else if (choice == 3)
{
    // Действие 3
}
else
{
    // Действие по умолчанию
}
```

**Способ 2: Оператор switch (классический)**

```csharp
switch (choice)
{
    case 1:
        // Действие 1
        break;
    case 2:
        // Действие 2
        break;
    case 3:
        // Действие 3
        break;
    default:
        // Действие по умолчанию
        break;
}
```

**Способ 3: Switch-выражение (C# 8.0+)**

```csharp
string result = choice switch
{
    1 => "Действие 1",
    2 => "Действие 2",
    3 => "Действие 3",
    _ => "По умолчанию"
};
```

#### Когда использовать

**Используйте switch:**
- Когда есть много вариантов (≥ 3)
- При работе с перечислениями (enum)
- Когда все варианты проверяют одну переменную на равенство

**Используйте if-else if:**
- Когда условия сложные (диапазоны, неравенства)
- Когда проверяются разные переменные
- Когда вариантов мало (2-3)

#### Практический пример (из Задачи 12)

```csharp
// Выбор формулы для вычисления площади
double CalculateArea(int figureType, double r)
{
    return figureType switch
    {
        1 => r * r,                          // Квадрат
        2 => Math.PI * r * r,                // Круг
        3 => (r * r * Math.Sqrt(3)) / 4,     // Треугольник
        _ => throw new ArgumentException("Неверный тип фигуры")
    };
}
```

#### Преимущества switch

1. **Читаемость** — явно видна структура выбора
2. **Производительность** — компилятор может оптимизировать (jump table)
3. **Безопасность** — компилятор проверяет полноту покрытия (для enum)
4. **Maintainability** — легко добавлять новые варианты

#### Недостатки switch

1. Работает только с константами и некоторыми типами
2. Не поддерживает сложные условия (диапазоны в старых версиях C#)
3. Требует break (в классическом switch)

### Трудности и их решение

1. **Выбор признака конца ввода:**
   - **Проблема:** сложность выбора подходящего специального значения
   - **Решение:** выбор значения вне диапазона реальных данных (например, -1 для оценок 2-5)

2. **Валидация множественных значений:**
   - **Проблема:** необходимость проверки каждого из нескольких вводимых значений
   - **Решение:** использование циклов для проверки всех элементов; флаги для отслеживания валидности

3. **Обработка некорректного ввода:**
   - **Проблема:** программа падает при вводе нечисловых данных
   - **Решение:** использование `TryParse` вместо `Parse` для безопасного преобразования

4. **Точность вещественных вычислений:**
   - **Проблема:** ошибки округления при сравнении вещественных чисел
   - **Решение:** использование порога сравнения (epsilon = 10⁻³)
