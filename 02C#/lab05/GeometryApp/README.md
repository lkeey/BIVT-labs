## Описание классов

### Базовый класс GeometricShape (абстрактный)

**Назначение:** Определяет общий интерфейс и базовые свойства для всех геометрических фигур.

**Protected поля:**
- `double centerX` - координата X центра фигуры
- `double centerY` - координата Y центра фигуры
- `string color` - цвет фигуры
- `double size` - размер фигуры (значение зависит от типа)
- `string shapeName` - название фигуры

**Публичные свойства:**
- `CenterX`, `CenterY`, `Color`, `Size` - с get/set
- `ShapeName` - только get (заполняется в конструкторе наследника)

**Абстрактные методы (требуют реализации в наследниках):**
- `abstract double CalculateArea()` - расчет площади фигуры
- `abstract void Draw()` - текстовое представление фигуры
- `abstract bool ContainsPoint(double x, double y)` - проверка принадлежности точки

**Виртуальный метод:**
- `virtual void SelectShape()` - вывод полной информации о фигуре (может быть переопределен)

### Класс Square : GeometricShape (Квадрат)

**Поле size:** длина стороны квадрата

**Реализация методов:**
- `CalculateArea()`: площадь = сторона²
- `Draw()`: выводит "□ Квадрат - центр: (x, y), сторона: s, цвет: color"
- `ContainsPoint(x, y)`: точка принадлежит если |x - centerX| ≤ size/2 && |y - centerY| ≤ size/2

**Пример создания:**
```csharp
Square square = new Square(50, 50, 30, "Красный");
double area = square.CalculateArea();  // 900.0
bool contains = square.ContainsPoint(55, 60);  // true
```

### Класс Circle : GeometricShape (Круг)

**Поле size:** диаметр круга

**Реализация методов:**
- `CalculateArea()`: площадь = π × (диаметр/2)²
- `Draw()`: выводит "○ Круг - центр: (x, y), диаметр: d, цвет: color"
- `ContainsPoint(x, y)`: точка принадлежит если расстояние от центра ≤ радиус

**Пример создания:**
```csharp
Circle circle = new Circle(100, 100, 40, "Синий");
double area = circle.CalculateArea();  // 1256.64
bool contains = circle.ContainsPoint(110, 110);  // true
```

### Класс Triangle : GeometricShape (Треугольник)

**Поле size:** высота равностороннего треугольника

**Реализация методов:**
- `CalculateArea()`: площадь = (высота² × √3) / 4
- `Draw()`: выводит "△ Треугольник - центр: (x, y), высота: h, цвет: color"
- `ContainsPoint(x, y)`: упрощенная проверка через радиус описанной окружности

**Пример создания:**
```csharp
Triangle triangle = new Triangle(75, 75, 25, "Зеленый");
double area = triangle.CalculateArea();  // 270.63
bool contains = triangle.ContainsPoint(80, 80);  // true
```

## Демонстрация полиморфизма

Ключевая особенность реализации - использование полиморфизма для работы с разнородной коллекцией объектов:

```csharp
List<GeometricShape> shapes = new List<GeometricShape>();

// Полиморфное добавление объектов разных классов
shapes.Add(new Square(10, 10, 20, "Красный"));
shapes.Add(new Circle(50, 50, 30, "Синий"));
shapes.Add(new Triangle(100, 100, 25, "Зеленый"));

// Полиморфный вызов методов - вызывается реализация конкретного класса
foreach (GeometricShape shape in shapes)
{
    shape.Draw();  // Вызов переопределенного метода Draw()
    Console.WriteLine($"Площадь: {shape.CalculateArea():F2}");  // Вызов CalculateArea()
    shape.SelectShape();  // Вызов виртуального метода
}
```

Во время выполнения CLR определяет фактический тип объекта и вызывает соответствующую реализацию метода из класса-наследника, даже если переменная имеет тип базового класса.

## Ключевые концепции ООП

### 1. Наследование
```csharp
public class Square : GeometricShape
```
Класс Square наследует все поля, свойства и методы базового класса GeometricShape.

### 2. Абстрактные классы и методы
```csharp
public abstract class GeometricShape
{
    public abstract double CalculateArea();
}
```
Нельзя создать экземпляр GeometricShape напрямую. Наследники обязаны реализовать все абстрактные методы.

### 3. Переопределение методов
```csharp
public override double CalculateArea()
{
    return size * size;  // Реализация для квадрата
}
```
Каждый наследник предоставляет свою реализацию абстрактных методов.

### 4. Полиморфизм
```csharp
List<GeometricShape> shapes = new List<GeometricShape>();
shapes.Add(new Square(...));
shapes.Add(new Circle(...));

foreach (var shape in shapes)
{
    shape.CalculateArea();  // Вызывается правильная реализация!
}
```
Во время выполнения вызывается метод конкретного класса, даже если переменная имеет тип базового класса.

### 5. Инкапсуляция
- Protected поля доступны наследникам, но скрыты снаружи
- Публичные свойства обеспечивают контролируемый доступ
- Абстрактные методы определяют контракт для наследников

## Преимущества полиморфизма

1. **Единый интерфейс**: работа с разными фигурами через общий тип `GeometricShape`
2. **Расширяемость**: легко добавить новые фигуры (Прямоугольник, Эллипс) без изменения существующего кода
3. **Гибкость**: коллекция `List<GeometricShape>` может хранить объекты любых классов-наследников
4. **Читаемость**: код становится более понятным и структурированным
