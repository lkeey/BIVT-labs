### 3. Реализация классов предметной области

**Задачи:**
- Реализовать абстрактный базовый класс `GeometricShape`:

  - Public свойства для доступа к полям
  - Абстрактные методы: `CalculateArea()`, `Draw()`, `ContainsPoint()`
  - Виртуальный метод `SelectShape()`

- Реализовать класс `Square`:
  - Конструктор с параметрами
  - Расчет площади по формуле S = a²
  - Проверка попадания точки в квадрат

- Реализовать класс `Circle`:
  - Конструктор с параметрами
  - Расчет площади по формуле S = π·r²
  - Проверка попадания точки через расстояние до центра

- Реализовать класс `Triangle`:
  - Конструктор с параметрами
  - Расчет площади равностороннего треугольника S = (a²·√3)/4
  - Проверка попадания точки

**Результат:** Созданы 4 класса (1 базовый + 3 производных) с полной реализацией функциональности.

## ВЫВОД

	В ходе выполнения лабораторной работы я успешно освоил ключевые механизмы объектно-ориентированного программирования на языке C#: наследование, полиморфизм, абстрактные классы и методы. Получил практический опыт создания иерархии классов с использованием абстрактного базового класса, реализации виртуальных и абстрактных методов, работы с коллекциями полиморфных объектов. Научился правильно применять модификаторы доступа для обеспечения инкапсуляции данных, использовать ключевое слово override для переопределения методов базового класса, работать с защищенными (protected) полями, которые доступны только наследникам.
	Реализовал полную иерархию классов геометрических фигур с базовым абстрактным классом GeometricShape и тремя производными классами: Square (квадрат), Circle (круг) и Triangle (треугольник). Класс GeometricShape инкапсулирует общие для всех фигур свойства (координаты центра, цвет, размер, название) и объявляет абстрактные методы, которые должны быть реализованы в каждом производном классе: CalculateArea() для расчета площади, Draw() для отображения информации, ContainsPoint() для проверки принадлежности точки фигуре. Каждый производный класс реализует собственную специфичную логику: квадрат использует формулу S = a², круг вычисляет площадь по формуле S = π·r², треугольник применяет формулу для равностороннего треугольника S = (a²·√3)/4. Освоил работу с математическими функциями Math.PI, Math.Sqrt, Math.Pow для выполнения геометрических вычислений.
	Научился работать с полиморфными коллекциями типа List<GeometricShape>, где хранятся объекты разных производных классов через ссылку на базовый тип. Понял механизм позднего связывания (late binding), когда при вызове метода на объекте базового типа автоматически вызывается переопределенная версия этого метода из конкретного производного класса. Освоил использование оператора is для проверки типа объекта во время выполнения программы, что позволяет безопасно выполнять операции, специфичные для конкретного типа фигуры. Применил паттерн проектирования "Шаблонный метод" через определение абстрактных методов в базовом классе, которые реализуются в производных классах по-разному, обеспечивая гибкость и расширяемость системы.
	Создал полнофункциональное графическое приложение на базе кроссплатформенного UI-фреймворка Avalonia, которое демонстрирует визуализацию геометрических фигур на холсте Canvas. Разработал XAML-разметку интерфейса с использованием Grid для разделения на две области: панель управления слева с кнопками, текстовыми полями и информационными метками, и Canvas справа для отрисовки фигур. Реализовал code-behind логику с обработчиками событий для создания случайных фигур, клика мыши по Canvas для выбора фигуры, проверки принадлежности точки с заданными координатами выбранной фигуре. Научился работать с визуальными элементами Avalonia: Rectangle для отрисовки квадратов, Ellipse для кругов, Polygon для треугольников, применять к ним заливку (Fill) и обводку (Stroke) с использованием объектов IBrush и SolidColorBrush. Освоил методы позиционирования элементов на Canvas через Canvas.SetLeft() и Canvas.SetTop().
	Реализовал функциональность сохранения и загрузки состояния приложения в текстовый файл с использованием классов StreamWriter и File для работы с файловой системой. Применил обработку исключений через конструкцию try-catch для корректной обработки ошибочных ситуаций: отсутствие файла, некорректный формат данных, проблемы с доступом к файлу. Научился правильно организовывать структуру данных для сериализации объектов в текстовый формат, используя символ разделителя для хранения свойств каждой фигуры в одной строке. Освоил десериализацию данных через разбор строк методом Split() и восстановление объектов нужного типа на основе сохраненной информации о типе фигуры.
	Полученные навыки работы с наследованием, полиморфизмом и абстрактными классами являются фундаментальными для объектно-ориентированного программирования и создания масштабируемых приложений. Понял принципы проектирования классов с учетом общности и специфики объектов, научился выделять общий функционал в базовый класс и реализовывать специфичное поведение в производных классах. Освоил применение принципа открытости/закрытости (Open/Closed Principle): система открыта для расширения через добавление новых производных классов, но закрыта для модификации базового класса. Эти знания и практические навыки станут основой для разработки сложных объектно-ориентированных систем, применения паттернов проектирования, работы с интерфейсами и создания архитектурно правильных приложений на C#.
