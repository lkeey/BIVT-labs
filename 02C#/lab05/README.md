# Лабораторная работа №5: Механизмы наследования и полиморфизма в C#

## Задание
**Продвинутый уровень (+4 балла): "Геометрические фигуры"**

Разработать иерархию классов геометрических фигур с демонстрацией наследования, полиморфизма и работой с абстрактными классами. Реализовать графический интерфейс для визуализации фигур на холсте.

## Цель работы
Освоение механизмов наследования и полиморфизма в C#: создание базового абстрактного класса, разработка производных классов, использование виртуальных и абстрактных методов, работа с коллекциями объектов базового типа.

## Структура проекта

### GeometryApp (консольная версия)
Первоначальная реализация с консольным интерфейсом и текстовым представлением фигур.

### GeometryGUI (графический интерфейс)
Графическая версия с визуализацией фигур на Canvas.

#### Иерархия классов

**GeometricShape.cs** - абстрактный базовый класс:
- Свойства: `CenterX`, `CenterY`, `Color`, `Size`, `ShapeName`
- Абстрактные методы: `CalculateArea()`, `Draw()`, `ContainsPoint()`
- Виртуальный метод: `SelectShape()`

**Square.cs** - класс квадрата:
- Дополнительное свойство: `Side`
- Реализация расчета площади: S = a²
- Проверка попадания точки в квадрат

**Circle.cs** - класс круга:
- Дополнительное свойство: `Diameter`
- Реализация расчета площади: S = π·r²
- Проверка попадания точки в круг

**Triangle.cs** - класс равностороннего треугольника:
- Дополнительное свойство: `Height`
- Реализация расчета площади: S = (a²·√3)/4
- Проверка попадания точки в треугольник

#### Графический интерфейс

**Левая панель управления:**
- Создание случайных фигур
- Очистка холста
- Информация о выбранной фигуре
- Проверка принадлежности точки фигуре
- Сохранение/загрузка фигур в файл

**Правая панель (Canvas):**
- Визуализация геометрических фигур
- Выбор фигуры кликом мыши
- Интерактивное взаимодействие

## Демонстрация ООП концепций

### 1. Наследование
Все фигуры наследуются от базового класса `GeometricShape`:
```csharp
public class Square : GeometricShape { ... }
public class Circle : GeometricShape { ... }
public class Triangle : GeometricShape { ... }
```

### 2. Абстрактные классы и методы
Базовый класс определяет контракт для всех фигур:
```csharp
public abstract class GeometricShape
{
    public abstract double CalculateArea();
    public abstract void Draw();
    public abstract bool ContainsPoint(double x, double y);
}
```

### 3. Полиморфизм
Работа с коллекцией разных фигур через единый интерфейс:
```csharp
List<GeometricShape> shapes = new List<GeometricShape>();
shapes.Add(new Square(...));
shapes.Add(new Circle(...));
shapes.Add(new Triangle(...));

foreach (var shape in shapes)
{
    shape.Draw();  // Вызывается соответствующая реализация
}
```

### 4. Инкапсуляция
Использование модификаторов доступа `protected` для полей базового класса и `public` для свойств.

## Вывод

В ходе выполнения лабораторной работы были освоены ключевые механизмы объектно-ориентированного программирования: наследование, полиморфизм, абстрактные классы и методы.

Разработана иерархия классов геометрических фигур с общим базовым классом `GeometricShape` и тремя производными классами: `Square`, `Circle`, `Triangle`. Каждый класс реализует собственную логику расчета площади и проверки принадлежности точки.

Продемонстрирован полиморфизм через работу с коллекцией `List<GeometricShape>`, где объекты разных типов обрабатываются единообразно, но выполняют специфичные для своего типа операции.

Реализованы две версии приложения: консольная для отработки логики классов и графическая с Canvas для визуализации фигур. Графическая версия демонстрирует интеграцию объектной модели с современным GUI фреймворком Avalonia.

Получен практический опыт применения абстракции для определения общего интерфейса, наследования для переиспользования кода, и полиморфизма для гибкой работы с разнородными объектами. Эти концепции составляют основу объектно-ориентированного проектирования и разработки масштабируемых приложений.
