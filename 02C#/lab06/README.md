# Лабораторная работа №6: Делегаты и события в C#

## Задание

Разработать приложение "Менеджер задач" с демонстрацией:
- Собственных делегатов (TaskFilter, NotifyHandler)
- Multicast-делегатов для системы уведомлений
- Лямбда-выражений для фильтрации и сортировки
- Стандартных обобщенных делегатов (Action, Func, Predicate)
- Событий (паттерн издатель-подписчик)
- LINQ для статистики и фильтрации

## Цель работы

Приобрести навыки работы с делегатами, событиями, анонимными методами, лямбда-выражениями и стандартными обобщёнными делегатами в C#.

## Структура проекта

### Классы предметной области

**TaskItem.cs** - модель задачи:
- `Name` - название задачи
- `Priority` - приоритет (1-5)
- `IsDone` - статус выполнения
- `CreatedAt` - время создания

**TaskEventArgs.cs** - аргументы события:
- `Task` - задача
- `Action` - тип действия (Добавлена/Удалена/Выполнена)
- `Time` - время события

**TaskManager.cs** - логика управления задачами:
- `Event TaskChanged` - событие изменения задач
- `AddTask()` - добавление задачи
- `RemoveTask()` - удаление задачи
- `ToggleTaskStatus()` - переключение статуса
- `GetFilteredTasks()` - фильтрация через Func<TaskItem, bool>

### Графический интерфейс (Avalonia UI)

**MainWindow.axaml/cs** - главное окно приложения:
- Панель добавления задач
- Список задач с фильтрацией
- Журнал событий (multicast-делегат)
- Статистика (Func/Action)
- Элементы управления (кнопки, CheckBox)

## Демонстрируемые концепции

### 1. Собственные делегаты
```csharp
delegate bool TaskFilter(TaskItem task);
delegate void NotifyHandler(string message);
```

### 2. Multicast-делегат
```csharp
notifyHandlers = LogToListBox;
notifyHandlers += ShowInStatusBar;
notifyHandlers("Сообщение"); // Вызовутся оба метода
```

### 3. Лямбда-выражения
```csharp
var sorted = allTasks
    .Where(t => filter(t))
    .OrderBy(t => t.IsDone)
    .ThenByDescending(t => t.Priority)
    .ToList();
```

### 4. Стандартные делегаты
```csharp
Func<int> getTotal = () => tasks.Count;
Action<string> updateLabel = text => labelStats.Text = text;
```

### 5. События
```csharp
public event EventHandler<TaskEventArgs> TaskChanged;

// Подписка
taskManager.TaskChanged += OnTaskChanged;

// Генерация
TaskChanged?.Invoke(this, new TaskEventArgs(task, "Добавлена"));
```

### 6. LINQ
```csharp
double avgPriority = tasks.Average(t => (double)t.Priority);
var byPriority = tasks.GroupBy(t => t.Priority).Select(g => ...);
```

## Использование

1. **Добавление задачи**: Введите название, выберите приоритет (1-5), нажмите "Добавить"
2. **Фильтрация**: Выберите фильтр из выпадающего списка
3. **Отметка выполненной**: Выберите задачу, нажмите "Выполнено/Вернуть"
4. **Удаление**: Выберите задачу, нажмите "Удалить"
5. **Журнал**: Все действия логируются в правую панель
6. **Статистика**: Отображается внизу с подробной информацией
7. **Уведомления**: Можно включать/выключать через CheckBox

## Особенности реализации

- Использование собственных делегатов для демонстрации базовых принципов
- Multicast-делегат вызывает несколько методов последовательно
- Паттерн "издатель-подписчик" через события
- Автоматическая сортировка: сначала невыполненные, затем по приоритету
- LINQ для сложных запросов и статистики
- Возможность динамической подписки/отписки от событий

## Преимущества делегатов

- Гибкость: можно менять поведение без изменения кода
- Переиспользование: один метод, разные реализации
- Читаемость: лямбда-выражения делают код компактным
- Расширяемость: легко добавить новые фильтры или обработчики
