## ПЛАН РАБОТЫ

### 1. Изучение теоретических основ делегатов

**Задачи:**
- Изучить понятие делегата как типа данных, хранящего ссылку на метод
- Понять концепцию сигнатуры метода и совместимости по сигнатуре
- Разобрать синтаксис объявления делегатов с ключевым словом delegate
- Освоить создание экземпляров делегатов и вызов методов через них

**Результат:** Получены базовые знания о делегатах как механизме передачи методов в качестве параметров.

---

### 2. Освоение multicast-делегатов

**Задачи:**
- Понять концепцию списка вызовов (invocation list)
- Изучить операторы += и -= для добавления/удаления методов
- Разработать систему уведомлений с несколькими обработчиками
- Реализовать multicast-делегат для одновременной записи в журнал и статус-бар

**Результат:** Создана система множественных уведомлений через один делегат.

---

### 3. Работа с лямбда-выражениями

**Задачи:**
- Изучить синтаксис лямбда-оператора => ("goes to")
- Освоить краткую форму записи анонимных методов
- Сравнить три способа: именованный метод, анонимный метод, лямбда
- Применить лямбда-выражения для фильтрации и сортировки задач

**Результат:** Реализована компактная фильтрация через лямбда-выражения.

---

### 4. Изучение стандартных делегатов

**Задачи:**
- Освоить Action<T> для методов без возвращаемого значения
- Изучить Func<T, TResult> для методов, возвращающих значение
- Понять Predicate<T> для логических условий
- Заменить собственные делегаты на стандартные обобщённые

**Результат:** Применены встроенные делегаты вместо собственных объявлений.

### 5. Реализация событий

**Задачи:**
- Изучить паттерн "издатель-подписчик"
- Создать класс TaskEventArgs как наследник EventArgs
- Объявить событие с ключевым словом event
- Реализовать генерацию событий через Invoke
- Организовать подписку и отписку от событий

**Результат:** Создана система событий для уведомления о изменениях задач.

**Задачи:**

- Использовать Count, Average для агрегатных функций

- Реализовать расширенную статистику через LINQ

**Результат:** Создана подробная статистика с группировкой по приоритетам.

## ВЫВОД

	В ходе выполнения лабораторной работы я успешно освоил ключевые механизмы работы с делегатами и событиями в языке программирования C#. Получил практический опыт создания собственных делегатов с использованием ключевого слова delegate, понял концепцию сигнатуры метода и совместимости типов. Научился объявлять делегаты как типы данных, создавать их экземпляры, записывать в них ссылки на методы и вызывать эти методы через переменные-делегаты. Освоил преимущества делегатов перед традиционными подходами с множественными условными операторами: делегаты позволяют менять поведение программы динамически, подставляя разные методы в одну и ту же переменную, что делает код более гибким и расширяемым.
	Реализовал multicast-делегаты, которые содержат не один, а несколько методов в списке вызовов (invocation list). Научился использовать операторы += для добавления методов в цепочку и -= для их удаления. Создал систему уведомлений, где один вызов multicast-делегата запускает последовательное выполнение нескольких обработчиков: запись в журнал событий с указанием времени и обновление статусной строки. Понял практическую ценность multicast-делегатов для реализации множественных реакций на одно действие, что особенно удобно в системах с подписками на события.
	Освоил работу с лямбда-выражениями как наиболее компактной и современной формой записи анонимных методов. Изучил синтаксис лямбда-оператора => (читается "goes to"), научился писать лямбда-выражения с одним выражением в теле и с блоком операторов. Сравнил три подхода к созданию делегатов: именованный метод (традиционный подход), анонимный метод через ключевое слово delegate (промежуточная форма) и лямбда-выражение (современный стандарт). Применил лямбда-выражения для создания компактных фильтров задач, функций подсчета статистики и преобразования данных. Понял, что лямбда-выражения делают код более читаемым за счет размещения логики непосредственно в месте использования, без необходимости создавать отдельные именованные методы для простых операций.
	Изучил стандартные обобщённые делегаты из библиотеки .NET: Action<T> для методов без возвращаемого значения, Func<T, TResult> для методов, возвращающих результат, и Predicate<T> для логических условий. Понял принцип работы с параметрами типа в угловых скобках: у Func последний параметр всегда определяет тип результата, а все предыдущие - типы входных параметров. Успешно заменил собственные объявления делегатов TaskFilter и NotifyHandler на стандартные Func<TaskItem, bool> и Action<string>, убедившись, что функциональность приложения не изменилась. Освоил преимущество стандартных делегатов: нет необходимости объявлять собственные типы для типовых случаев, что упрощает код и делает его более универсальным.
	Реализовал механизм событий и паттерн проектирования "издатель-подписчик" (publisher-subscriber). Создал класс TaskEventArgs как наследник базового класса EventArgs для передачи данных о событии: информации о задаче, типе действия и времени события. Объявил событие с ключевым словом event, понял ключевое отличие события от обычного публичного делегата: событие можно вызвать только изнутри класса, в котором оно объявлено, что обеспечивает инкапсуляцию и защиту от некорректного использования. Организовал подписку на события через оператор += и отписку через -=, реализовал возможность динамического управления подписками через CheckBox "Включить уведомления". Генерировал события через оператор ?. для безопасной проверки наличия подписчиков и метод Invoke для передачи данных всем подписанным обработчикам.
	Освоил технологию LINQ (Language Integrated Query) для работы с коллекциями через декларативные запросы. Применил методы Where для фильтрации задач по условию, OrderBy и ThenByDescending для многоуровневой сортировки (сначала по статусу, затем по приоритету), Count для подсчета количества элементов с условием, Average для вычисления среднего значения. Использовал GroupBy для группировки задач по приоритету и Select для преобразования групп в строковое представление. Реализовал расширенную статистику, показывающую не только количество задач, но и средний приоритет, процент выполненных задач, распределение по уровням приоритета. Понял, что все методы LINQ принимают делегаты (в основном Func<T, TResult>) и активно используют лямбда-выражения для передачи логики фильтрации, преобразования и сортировки.
	Полученные навыки работы с делегатами, событиями и лямбда-выражениями являются фундаментальными для современного программирования на C#. Понял преимущества делегатов перед традиционными условными конструкциями: гибкость через динамическую подмену логики, переиспользование кода через единый интерфейс вызова, расширяемость через легкое добавление новых обработчиков без изменения существующего кода. Освоил паттерн "издатель-подписчик" как ключевой механизм слабой связанности компонентов, который широко применяется в GUI-фреймворках, реактивном программировании и микросервисных архитектурах. Научился писать компактный и выразительный код с помощью лямбда-выражений и LINQ, что повышает читаемость и поддерживаемость программ. Эти знания и практические навыки станут основой для работы с асинхронным программированием (async/await), реактивными расширениями (Rx), функциональным программированием и создания масштабируемых event-driven систем на платформе .NET.
